1. OVERVIEW: WHAT THE XAPP DOES
================================================================================

The Energy Saving xApp is a **Zero-Touch Energy Saving with Cell Utilization** application that:

1. **MONITORS** cell utilization and UE SINR metrics via E2SM KPM
2. **ANALYZES** which cells have low utilization (≤1 UE)
3. **DECIDES** to shut down underutilized cells for energy savings
4. **EXECUTES** handovers to move UEs from cells marked for shutdown
5. **CONTROLS** the RAN via E2SM RC (RAN Control) to switch off cells

================================================================================
2. STEP-BY-STEP XAPP ALGORITHM
================================================================================

2.1 PHASE 1: DATA COLLECTION (Every 100ms)
-------------------------------------------
- Subscribes to E2SM KPM (RAN Function ID = 2)
- Receives SINR measurements for all UEs in all cells
- Builds a data structure: cells_sinr_map[MAX_REGISTERED_CELLS]
- Tracks: cellID, connected UEs, serving SINR, neighbor SINRs

2.2 PHASE 2: CELL EVALUATION (Every 5 seconds)
-----------------------------------------------
```c
// First pass: Mark cells for shutdown
if (cell->numOfConnectedUEs <= 1) {
    cell->pending_shutdown = true;
    cell->shutdown_start_time = time(NULL);
    printf("Cell %d marked for shutdown\n", cell->cellID);
}
```

2.3 PHASE 3: HANDOVER EXECUTION
-------------------------------
For each cell marked for shutdown:
1. **Find Target Cell** for each UE:
   ```c
   // Find best neighbor cell with SINR > -10 dB
   if (data.neighCells[i].sinr > MIN_SINR && 
       data.neighCells[i].sinr > max_sinr) {
       max_sinr = data.neighCells[i].sinr;
       targetCell = i;
   }
   ```

2. **Trigger Handover** via E2SM RC:
   ```c
   // Send handover control message
   control_sm_xapp_api(&node->id, SM_RC_ID, &rc_ctrl);
   ```

2.4 PHASE 4: CELL SHUTDOWN
--------------------------
After all handovers complete (or timeout after 30s):
```c
// Send cell switch-off control message
control_sm_xapp_api(&node->id, SM_RC_ID, &rc_ctrl);
```

================================================================================
3. WHAT METRICS DOES THE XAPP USE?
================================================================================

3.1 PRIMARY METRICS (For Decision Making):
------------------------------------------
- **L3servingSINR3gpp_cell_{cellID}_UEID_{ueID}**: Serving cell SINR
- **L3neighSINRListOf_UEID_{ueID}_of_Cell_{cellID}**: Neighbor cell SINRs

3.2 SECONDARY METRICS (For Logging/Monitoring):
-----------------------------------------------
- **RRU.PrbTotDl/Ul**: Resource utilization
- **DRB.PdcpSduVolumeDL/UL**: Throughput
- **DRB.UEThpDl/Ul**: UE throughput
- **DRB.RlcSduDelayDl**: Latency

3.3 DECISION CRITERIA:
---------------------
- **Cell Shutdown**: numOfConnectedUEs ≤ 1
- **Handover Target**: SINR > -10 dB (MIN_SINR)
- **Timeout**: 30 seconds for handover completion

================================================================================
4. HOW DOES THE XAPP USE E2SM RC (RAN CONTROL)?
================================================================================

4.1 RC CONTROL MESSAGES SENT:
-----------------------------

A) **HANDOVER CONTROL MESSAGE**:
```c
// Header
rc_ctrl.hdr = gen_rc_ctrl_hdr(FORMAT_1_E2SM_RC_CTRL_HDR, ue_id, 
                              CONNECTED_MODE_MOBILITY, HANDOVER_CONTROL_7_6_4_1);

// Message contains:
// 1. Target Primary Cell ID (NR CGI or E-UTRA CGI)
// 2. List of PDU sessions for handover
// 3. List of DRBs for handover  
// 4. List of Secondary cells to be setup
```

B) **CELL SHUTDOWN CONTROL MESSAGE**:
```c
// Header  
rc_ctrl.hdr = gen_rc_ctrl_hdr(FORMAT_1_E2SM_RC_CTRL_HDR, ue_id,
                              ENERGY_STATE, CELL_OFF);

// Message contains:
// 1. Target Primary Cell ID (cell to switch off)
```

4.2 RC CONTROL PARAMETERS:
--------------------------
- **RAN Function ID**: 3 (SM_RC_ID)
- **Control Action ID**: 
  - HANDOVER_CONTROL_7_6_4_1 (for handovers)
  - CELL_OFF (for cell shutdown)
- **RIC Style Type**:
  - CONNECTED_MODE_MOBILITY (for handovers)
  - ENERGY_STATE (for cell shutdown)

6. EXAMPLE XAPP EXECUTION FLOW
================================================================================

6.1 INITIAL STATE:
-----------------
- Cell 1: 3 UEs connected
- Cell 2: 1 UE connected  
- Cell 3: 0 UEs connected

6.2 XAPP DECISION:
-----------------
- Cell 2: numOfConnectedUEs = 1 ≤ 1 → Mark for shutdown
- Cell 3: numOfConnectedUEs = 0 ≤ 1 → Mark for shutdown

6.3 HANDOVER EXECUTION:
----------------------
For Cell 2 (1 UE):
1. Check neighbor SINRs for UE
2. Find best target (e.g., Cell 1 with SINR = 5 dB)
3. Send handover control: "Move UE from Cell 2 to Cell 1"
4. Wait for handover completion

6.4 CELL SHUTDOWN:
-----------------
After handover completes:
1. Send cell shutdown control: "Switch off Cell 2"
2. Cell 2 goes to sleep mode
3. Energy saved! ⚡
